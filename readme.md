## 💡 Key Learnings

### Problem-Solving Approach
1. **Understand the problem** - Read examples, identify constraints
2. **Think of approach** - Brute force, optimized, patterns
3. **Analyze complexity** - Time and space trade-offs
4. **Code & test** - Implement and verify with examples
5. **Optimize** - Find better solutions if time/space permits

### Optimization Patterns
- **Two Pointer Technique** - Used in array problems for O(n) solutions
- **Hash Maps** - Converting O(n²) to O(n) in frequency problems
- **Tail Recursion** - Reducing space complexity from O(n) to O(1)
- **In-place Modification** - Solving arrays without extra space
